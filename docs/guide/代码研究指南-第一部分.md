### 1.2 泄露机制详解

#### Source Map 原理

Source map 是一种映射文件，用于将编译/压缩后的代码映射回原始源代码，方便调试。

**正常流程：**
```
TypeScript 源码 → 编译 → JavaScript + Source Map
                    ↓
            开发者工具加载 Source Map
                    ↓
            浏览器/IDE 显示原始源码
```

**泄露流程：**
```
1. Claude Code 构建过程中生成 Source Map
2. Source Map 文件（cli.js.map）被打包进 npm 包
3. Source Map 中包含指向原始源码的 URL
4. 原始源码托管在 Anthropic 的 R2 存储桶
5. R2 存储桶配置允许公开访问
        ↓
    完整源码公开可下载
```

#### 技术细节

**Source Map 文件内容示例：**
```json
{
  "version": 3,
  "sources": [
    "src/QueryEngine.ts",
    "src/Tool.ts",
    "src/main.tsx",
    ... 1,900+ files
  ],
  "sourcesContent": [
    "export class QueryEngine { ... }",
    "export interface Tool { ... }",
    ...
  ],
  "mappings": "...",
  "sourceRoot": "https://anthropic-r2-bucket.com/src/"
}
```

**问题根源：**
1. **构建配置错误**：生产环境未禁用 source map 生成
2. **打包配置遗漏**：`.npmignore` 未排除 `.map` 文件
3. **存储桶配置错误**：源码存储桶允许公开访问
4. **发布流程缺陷**：缺乏预发布安全检查
